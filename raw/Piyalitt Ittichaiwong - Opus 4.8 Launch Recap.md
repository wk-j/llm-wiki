Anthropic จากบทความเปิดตัว Claude Opus 4.8
Note: ซึ่งตรงใจมากๆ เพราะตอนนี้ชอบเนียนว่าทำแล้วแต่ยังไม่ได้ 55555
ประโยคนี้คือหัวใจของการอัปเดตครั้งล่าสุด เพราะหนึ่งในการพัฒนาที่ Anthropic ชูว่าโดดเด่นที่สุดใน Claude Opus 4.8 ไม่ใช่แค่ความเก่งที่เพิ่มขึ้น แต่คือการที่โมเดลซื่อสัตย์มากขึ้น ยอมบอกว่าตัวเองไม่แน่ใจ และไม่รีบสรุปว่าทำสำเร็จทั้งที่หลักฐานยังไม่พอ
เมื่อวันที่ 28 พฤษภาคม 2026 Anthropic ประกาศอัปเกรด Claude Opus เป็นเวอร์ชันใหม่คือ Opus 4.8 ซึ่งต่อยอดจาก Opus 4.7 ทำคะแนนดีขึ้นในหลาย benchmark เป็นผู้ร่วมงานที่มีประสิทธิภาพมากขึ้น และที่สำคัญคือเปิดให้ใช้ในราคาเท่าเดิม
═══════════════════
🎯 Opus 4.8 ดีขึ้นตรงไหน
🔹 Opus 4.8 ต่อยอดจาก Opus 4.7 โดยทำได้ดีขึ้นในการทดสอบหลายด้าน ทั้ง coding งานแบบ agentic การใช้เหตุผล และงาน knowledge work ในทางปฏิบัติ โดยรายละเอียดการทดสอบและการเปรียบเทียบความสามารถฉบับเต็มอยู่ใน System Card ของรุ่นนี้
🔹 ผู้ทดสอบรุ่นแรกพบว่า Opus 4.8 น่าเชื่อถือขึ้นและมีดุลพินิจที่คมขึ้นเวลาทำงานแบบ agentic คือทำงานเป็นขั้นเป็นตอนแทนเราได้ดีกว่าเดิม โดยรวมแล้วเป็นการพัฒนาที่ Anthropic เรียกว่าไม่หวือหวามาก แต่จับต้องได้จริง
═══════════════════
💡 จุดเด่นหลักคือความซื่อสัตย์
🔹 Anthropic ฝึกโมเดลทุกตัวให้ซื่อสัตย์อยู่แล้ว เช่น ให้หลีกเลี่ยงการกล่าวอ้างในสิ่งที่ตัวเองหาหลักฐานมาสนับสนุนไม่ได้ แต่ปัญหาที่เจอบ่อยของ AI โดยทั่วไปคือมันมักรีบสรุปและกล่าวอ้างอย่างมั่นใจว่างานคืบหน้า ทั้งที่หลักฐานยังบางมาก
🔹 ผู้ทดสอบรุ่นแรกรายงานว่า Opus 4.8 มีแนวโน้มจะบอกความไม่แน่ใจเกี่ยวกับงานของตัวเองมากขึ้น และกล่าวอ้างในสิ่งที่ไม่มีหลักฐานรองรับน้อยลง ซึ่งสอดคล้องกับผลการประเมินภายในของ Anthropic
🔹 ตัวเลขที่ชัดเจนคือ Opus 4.8 มีโอกาสปล่อยให้ข้อบกพร่องใน code ที่ตัวเองเขียนผ่านไปโดยไม่ทักท้วง น้อยกว่ารุ่นก่อนหน้าราวสี่เท่า
═══════════════════
🔍 ผลการประเมินด้าน Alignment
🔹 ก่อนปล่อยรุ่นนี้ Anthropic ทำการประเมินด้าน Alignment อย่างละเอียดเหมือนทุกครั้ง โดยทีม Alignment สรุปว่า Opus 4.8 ทำสถิติสูงสุดใหม่ในมาตรวัดคุณลักษณะเชิงสังคม เช่น การสนับสนุนให้ผู้ใช้ตัดสินใจได้เอง และการกระทำเพื่อประโยชน์สูงสุดของผู้ใช้
🔹 ผลการประเมินยังพบว่า Opus 4.8 มีอัตราพฤติกรรมที่ไม่สอดคล้องกับเป้าหมาย เช่น การหลอกลวงหรือการให้ความร่วมมือกับการใช้งานในทางที่ผิด ต่ำกว่า Opus 4.7 อย่างมาก และอยู่ในระดับใกล้เคียงกับโมเดลที่ Alignment ดีที่สุดของบริษัทอย่าง Claude Mythos Preview
🔹 รายงานการประเมิน Alignment ฉบับเต็มพร้อมชุดการทดสอบความปลอดภัยก่อนนำไปใช้งานจริง อยู่ใน System Card ของ Opus 4.8
═══════════════════
🚀 ของใหม่ที่มาพร้อมกัน
🔹 effort control บน claude . ai และ Cowork คือปุ่มควบคุมใหม่ข้างตัวเลือกโมเดล ที่ให้เราเลือกได้ว่าจะให้ Claude ทุ่มความพยายามกับคำตอบมากแค่ไหน ถ้าตั้งระดับสูง Claude จะคิดบ่อยขึ้นและลึกขึ้นเพื่อให้คำตอบที่ดีกว่า ถ้าตั้งระดับต่ำ Claude จะตอบเร็วขึ้นและใช้ rate limit ช้าลง ฟีเจอร์นี้เปิดให้ใช้ทุกแพ็กเกจ
🔹 dynamic workflows เป็นฟีเจอร์ใหม่ใน Claude Code ที่อยู่ในขั้น research preview ทำให้ Claude รับงานขนาดใหญ่ขึ้นได้ โดยวางแผนงานเอง แล้วสั่ง subagents ทำงานคู่ขนานเป็นร้อยตัวในเซสชันเดียว ก่อนจะตรวจสอบผลลัพธ์ของตัวเองแล้วค่อยรายงานกลับ ตัวอย่างเช่น Claude Code กับ Opus 4.8 สามารถทำการ migrate code ระดับ codebase ที่มี code หลายแสนบรรทัดได้ตั้งแต่เริ่มจนถึงขั้น merge โดยใช้ test suite ที่มีอยู่เป็นเกณฑ์ตัดสิน ฟีเจอร์นี้เปิดให้แพ็กเกจ Enterprise, Team และ Max
🔹 Messages API ตอนนี้รองรับการใส่ system เข้าไปใน array ของ messages ได้แล้ว ทำให้ developer อัปเดตคำสั่งให้ Claude ระหว่างทำงานได้โดยไม่ทำให้ prompt cache พัง และไม่ต้องส่งผ่าน user turn สามารถใช้อัปเดตสิทธิ์การใช้งาน งบ token หรือข้อมูล environment ระหว่างที่ agent กำลังทำงานได้
🔹 fast mode ของ Opus 4.8 ที่โมเดลทำงานได้เร็วขึ้น 2.5 เท่า ตอนนี้ราคาถูกลงกว่ารุ่นก่อนหน้าถึงสามเท่า
═══════════════════
⚙️ เรื่อง effort ที่ควรรู้
🔹 โดยค่าเริ่มต้น Opus 4.8 ตั้งไว้ที่ระดับ high ซึ่ง Anthropic ประเมินว่าเป็นจุดสมดุลที่ดีที่สุดระหว่างคุณภาพกับประสบการณ์ใช้งาน สำหรับงาน coding ระดับนี้ใช้ token พอ ๆ กับค่าเริ่มต้นของ Opus 4.7 แต่ให้ผลลัพธ์ที่ดีกว่า
🔹 เราเลือกระดับ extra (เรียกว่า xhigh ใน Claude Code) หรือ max ได้ เพื่อให้โมเดลใช้ token มากขึ้นและได้ผลลัพธ์ที่ดีขึ้น โดยมีคำแนะนำให้ใช้ extra กับงานที่ยากและงานแบบ asynchronous ที่รันยาว
🔹 เพื่อรองรับการใช้ token ที่มากขึ้นในระดับ effort สูง Claude Code ได้เพิ่ม rate limit ให้แล้ว เราจึงเลือกระดับที่เหมาะกับโปรเจกต์ของเราได้
═══════════════════
🔮 ก้าวต่อไป
🔹 Anthropic บอกว่ายังมีงานที่ต้องทำต่อ โดยกำลังพัฒนาโมเดลที่ให้ความสามารถใกล้เคียง Opus แต่ในราคาที่ถูกลง
🔹 นอกจากนี้ยังมีแผนจะปล่อยโมเดลระดับใหม่ที่ฉลาดกว่า Opus โดยภายใต้ Project Glasswing ตอนนี้มีองค์กรจำนวนหนึ่งกำลังใช้ Claude Mythos Preview สำหรับงานด้านความปลอดภัยทางไซเบอร์อยู่แล้ว
🔹 โมเดลที่ความสามารถระดับนี้ต้องมีมาตรการป้องกันด้านไซเบอร์ที่แข็งแรงขึ้นก่อนจะเปิดให้ใช้ทั่วไป ซึ่งตอนนี้การพัฒนามาตรการเหล่านั้นคืบหน้าอย่างรวดเร็ว และคาดว่าจะนำโมเดลระดับ Mythos มาให้ลูกค้าทุกคนได้ภายในไม่กี่สัปดาห์ข้างหน้า
═══════════════════
💰 ราคาและการเปิดให้ใช้งาน
🔹 Claude Opus 4.8 เปิดให้ใช้งานทุกที่แล้วตั้งแต่วันนี้ โดยราคาการใช้งานปกติเท่ากับ Opus 4.7 คือ 5 ดอลลาร์ต่อหนึ่งล้าน input tokens และ 25 ดอลลาร์ต่อหนึ่งล้าน output tokens
🔹 ส่วน fast mode ราคาอยู่ที่ 10 ดอลลาร์ต่อหนึ่งล้าน input tokens และ 50 ดอลลาร์ต่อหนึ่งล้าน output tokens ฝั่ง developer เรียกใช้ผ่าน Claude API ด้วยชื่อโมเดล claude-opus-4-8
═══════════════════
ปิดท้าย ในยุคที่โมเดล AI แข่งกันที่ความเก่งและความเร็ว การที่ Anthropic เลือกชูเรื่องความซื่อสัตย์เป็นจุดเด่นของ Opus 4.8 สะท้อนทิศทางที่น่าสนใจ เพราะสำหรับงานจริงที่เราต้องนำผลลัพธ์ของโมเดลไปใช้ต่อ การที่ AI ยอมบอกว่าไม่แน่ใจ อาจมีค่ามากกว่าการที่มันตอบเร็วแต่มั่นใจผิด ๆ
