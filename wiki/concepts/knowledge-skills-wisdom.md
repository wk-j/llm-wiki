# Knowledge / Skills / Wisdom / ความรู้ ทักษะ ปัญญา

**[[matt-pocock|Matt Pocock]] (เคยเป็นครูสอนร้องเพลง 10 ปี ก่อนมาสอน dev) แยกสามอย่างที่ต้องมีถึงจะเก่งอะไรสักอย่าง: knowledge, skills, wisdom.** กรอบนี้เป็นรากของ [[teach-skill|teach skill]] ของเขา และอธิบายว่าทำไม "ไปอยู่ที่นั่นจริง ๆ" ถึงสำคัญ

## สามชั้น

1. **Knowledge (ความรู้)** — เข้าใจว่าสิ่งนั้นคืออะไร เก็บไว้ในหัว
2. **Skills (ทักษะ)** — ทำมาหลายครั้งจนเป็น muscle memory
3. **Wisdom (ปัญญา)** — รู้ว่าควรทำตอนไหน มันฟิตกับโลกจริงยังไง

## ทำไม wisdom ถึงได้มายาก

> "Wisdom is almost impossible to obtain without actually having done the thing in the exact context where you need to do it."
> (ปัญญาแทบเป็นไปไม่ได้ที่จะได้มาโดยไม่ได้ลงมือทำสิ่งนั้นในบริบทจริง ๆ ที่ต้องใช้)

knowledge กับ skills เรียนเอา/ฝึกเอาได้ แต่ wisdom ต้องลงไปทำในบริบทนั้นเอง. Matt ยกตัวอย่าง: อยากได้ wisdom แบบคนที่ [[anthropic|Anthropic]] — ความรู้กับทักษะหาได้ แต่ wisdom ต้อง **ไปอยู่ Anthropic จริง ๆ** ถึงจะเข้าใจว่าเขาทำกันยังไง

## ทำไมเรื่องนี้สำคัญในยุค AI

ของที่ตื่นเต้น: เราเอา **knowledge + skills (สองข้อแรก)** มามัดเป็น skill ที่ reuse และแจกทีมได้ — เหมือนดึง function ที่ซ้ำออกมาเป็น shared function. "ผมทำแผนแบบนี้มาร้อยครั้งจนรู้ว่าแผนที่ดีเป็นยังไง — ผมแปลงมันเป็น skill แจกทีม ทุกคน plan แบบเดียวกันและช่วยกันปรับ skill นั้น"

ผลคือ **ยกพื้น (raise the floor)** ของวิศวกรทั้งทีม. แต่ wisdom ยัง bundle ไม่ได้ — ส่วนนั้นยังต้องสะสมจากการลงมือทำเอง (เชื่อมกับ [[skill-atrophy]]: ถ้า delegate ทุกอย่างจนไม่เคยทำเอง ก็ไม่เกิด wisdom)

## ในตัว teach skill

[[teach-skill|teach skill]] ใช้กรอบนี้ตรง ๆ: เริ่มจาก `mission.md` (ผู้เรียนอยากทำอะไรในโลกจริง) → จัด knowledge เป็น linear path ผ่าน "ป่าความรู้" → ใช้ quiz เพิ่ม storage strength → ชี้ไป primary source. มันสอนตาม **zone of proximal development** (สอนของที่ "เอื้อมถึงได้ถ้ามีคนช่วย" ทีละขั้น)

> ได้อะไร: เวลาจะเรียน (หรือสอน AI/ทีม) อะไร แยกให้ออกว่ากำลังขาด knowledge, skills, หรือ wisdom — เพราะสามอย่างนี้ได้มาด้วยวิธีต่างกัน และ wisdom ลัดไม่ได้
