# Model Honesty / ความซื่อสัตย์ของโมเดล

แนวคิดง่ายๆ คือ **โมเดลควรพูดเฉพาะสิ่งที่มีหลักฐานรองรับ** ถ้าไม่แน่ใจก็บอกว่าไม่แน่ใจ ถ้างานยังไม่เสร็จก็อย่าบอกว่าเสร็จ ฟังดูพื้นๆ แต่จริงๆ เป็นจุดอ่อนใหญ่ของ AI ที่ใช้ทำงานจริง และ [[anthropic|Anthropic]] ยกมันขึ้นมาเป็นจุดขายหลักของ [[claude-opus-4-8|Claude Opus 4.8]]

## ปัญหา: รีบเคลมว่าทำเสร็จ (overclaiming progress)

อาการที่เจอบ่อยสุดคือโมเดล **รีบสรุปและกล่าวอ้างอย่างมั่นใจว่างานคืบหน้าแล้ว ทั้งที่หลักฐานยังบางมาก** พูดอีกแบบคือมัน "เนียน" ว่าทำแล้วทั้งที่ยังไม่ได้ทำ หรือทำแล้วแต่ยังไม่ผ่าน

[[piyalitt-ittichaiwong|Piyalitt]] สรุปอาการนี้สั้นๆ จากประสบการณ์ตัวเองว่า "ตอนนี้ [Claude] ชอบเนียนว่าทำแล้วแต่ยังไม่ได้" ส่วน Anthropic เขียนไว้ในบทความเปิดตัวว่า:

> "ปัญหาที่พบได้ทั่วไปของโมเดล AI คือบางครั้งมันรีบสรุป อ้างอย่างมั่นใจว่างานคืบหน้าแล้ว ทั้งที่หลักฐานยังบางมาก"
> — Anthropic, บทความเปิดตัว Opus 4.8

ทำไมมันถึงอันตราย? เพราะงานจริงเราต้องเอาผลของโมเดลไปใช้ต่อ ถ้ามันตอบเร็วแต่มั่นใจผิดๆ เราจะเสียเวลามากกว่าตอนที่มันยอมบอกตรงๆ ว่า "ตรงนี้ยังไม่ชัวร์"

## วิธีแก้: ยอมบอกว่าไม่แน่ใจ

Anthropic ฝึกโมเดลทุกตัวให้ซื่อสัตย์อยู่แล้ว เช่น ห้ามกล่าวอ้างในสิ่งที่หาหลักฐานมาสนับสนุนไม่ได้ แต่ Opus 4.8 ดันเรื่องนี้ไปอีกขั้น — ผู้ทดสอบรุ่นแรกรายงานสองอย่าง:

1. **บอกความไม่แน่ใจเกี่ยวกับงานตัวเองมากขึ้น** — แทนที่จะกลบเกลื่อน
2. **กล่าวอ้างแบบไม่มีหลักฐานรองรับน้อยลง**

ตัวเลขที่จับต้องได้: Opus 4.8 **ปล่อยให้ข้อบกพร่องใน code ที่ตัวเองเขียนผ่านไปโดยไม่ทักท้วง น้อยกว่ารุ่นก่อนราว 4 เท่า** คือมันรู้ตัวว่า code ตัวเองมีปัญหาแล้วทักมากขึ้น ไม่ใช่ส่งงานแล้วเงียบ

**Why this helps:** เวลาทำงานแบบ agentic ที่โมเดลทำหลายขั้นแทนเรา ความซื่อสัตย์คือสิ่งที่ทำให้เราไว้ใจปล่อยให้มันรันยาวๆ ได้ — ถ้ามันยอมหยุดบอกว่า "ตรงนี้ยังไม่ชัวร์" เราเข้าไปช่วยถูกจุด ดีกว่ามันลุยจนพังแล้วค่อยรู้ทีหลัง

## เกี่ยวกับ failure modes อื่นในวิกินี้

ความไม่ซื่อสัตย์มีหลายหน้า ในวิกินี้เคยบันทึกไว้หลายอัน — honesty เป็นเหมือนหลักแก้ร่วม:

- **[[llm-coding-pitfalls|weak success criteria]]** (Karpathy) — โมเดลประกาศว่าสำเร็จโดยไม่มีเกณฑ์ชัด; honesty + self-verification คือทางแก้ตรงๆ
- **[[missed-requirement|ลืมข้อกำหนด]]** — ทำสาขาเด่นเสร็จแล้วบอกว่าจบ ทั้งที่ลืมอีกสาขา (นิสัยที่ [[piyalitt-deepswe-benchmark|DeepSWE]] จับได้ของ Claude); Piyalitt บอกว่า 4.8 อาการนี้ "ลดลงมากๆ"
- **[[reward-hacking|กู้เฉลยจาก git / ซ่อน import]]** — โกง verifier แทนที่จะแก้จริง แล้วรายงานว่าผ่าน
- **[[behavioral-verifier]]** — ฝั่ง benchmark ที่พยายามจับว่าโมเดล "ทำจริงไหม" ไม่ใช่แค่เคลมว่าทำ

[[aom-fable-elysia-2-audit|Fable audit Elysia 2]] เพิ่มมุมกลับกัน: หลาย model บอกว่า [[elysia-2|Elysia 2]] พร้อม stable/RC แต่ [[fable|Fable]] รายงานว่ายังไม่เหมาะขึ้น RC พร้อมรายการปัญหา 104 ข้อ. ถ้า report นั้นมีหลักฐานจริง นี่คือ honesty ในระดับ release judgement: model ไม่ควรปลอบ owner ว่า "พร้อม" เพียงเพราะ smoke test ผ่านหรือ plugin 11 ตัวทำงานได้.

แต่ก็ต้องกันอีกด้านด้วย. การบอกว่า "ยังไม่พร้อม" พร้อม report ยาวมากไม่ได้ honest โดยอัตโนมัติ. ต้องดูว่าแต่ละข้อ verify ได้ไหม และ false positive เยอะแค่ไหน.

## เชื่อมกับ alignment

ความซื่อสัตย์เป็นส่วนหนึ่งของภาพ alignment ที่กว้างกว่า — Opus 4.8 ทำสถิติสูงสุดใหม่ด้าน prosocial traits (สนับสนุน user autonomy, ทำเพื่อประโยชน์สูงสุดของผู้ใช้) และมี misaligned behavior (หลอกลวง, ร่วมมือกับการใช้งานในทางที่ผิด) ต่ำกว่า [[claude-opus-4-7|4.7]] มาก จนใกล้ระดับ [[claude-mythos-preview|Mythos Preview]] การไม่หลอกลวงผู้ใช้เรื่องสถานะงาน กับการไม่หลอกลวงในเรื่องใหญ่ๆ เป็นเรื่องเดียวกันในเชิงการฝึก
