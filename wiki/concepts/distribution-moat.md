# Distribution Moat / คูเมืองด้าน Distribution

Distribution moat คือความได้เปรียบจากการถือช่องทางถึงผู้ใช้ ไม่ใช่แค่การมี product ที่ดี. ช่องทางนี้อาจเป็น default setting, browser, operating system, app ecosystem, brand habit, หรือ network ที่คนใช้อยู่ทุกวัน

## ทำไมสำคัญใน AI

พอ AI ทำให้ feature ถูก copy เร็วขึ้น moat ที่เป็น “เราทำ feature นี้ได้ก่อน” จะบางลง. สิ่งที่ copy ยากกว่าคือช่องทางที่พาผู้ใช้มาหา feature นั้น

ใน [[how-perplexity-lost-ai-war|How Perplexity Lost the AI War]] วิดีโอบอกว่า [[perplexity|Perplexity]] มี answer+citation ที่ดี แต่ [[google|Google]] ถือ search habit, Chrome, Android, YouTube, Gmail, Drive, และ ad surface. พอ Google เพิ่ม AI answer เข้า surface เดิม ผู้ใช้จำนวนมากไม่ต้องเปลี่ยนพฤติกรรม

ผลคือ: incumbent copy feature แล้ววางใน distribution เดิมได้เร็วกว่า startup สร้าง habit ใหม่

## เชื่อมกับ Value Migration from Code

[[value-migration-from-code|Value Migration from Code]] ของ [[thorsten-ball|Thorsten Ball]] บอกว่ามูลค่าซอฟต์แวร์จะย้ายจาก code ไปยัง data, permissions, distribution, trust, compliance และสินทรัพย์อื่นที่ agent copy ไม่ได้

Distribution moat คือหนึ่งในสินทรัพย์นั้น. ในโลกที่ code หรือ feature สร้างได้เร็วขึ้น ช่องทางถึงผู้ใช้กลายเป็นสิ่งที่แพงขึ้น ไม่ใช่ถูกลง

## ตัวอย่างจาก Perplexity

[[comet-browser|Comet]] คือความพยายามแก้ distribution moat โดยถือ browser เอง. แต่ Mondo Startups ชี้ว่า search/browser habit เหนียวมาก. Microsoft ยังเปลี่ยน Bing share ได้ยากแม้มี Windows และ ChatGPT integration

ได้อะไร: เวลาประเมิน AI startup อย่าถามแค่ “product ดีกว่าไหม” ให้ถามด้วยว่า “ผู้ใช้จะเจอและใช้มันเป็น default ได้อย่างไร”
