# Grounding / การอ้างอิงหลักฐาน

Grounding คือการทำให้ไอเดียหรือข้อเสนอของ AI agent มี "ฐาน" รองรับที่ชัดเจน ไม่ได้เกิดจากการคาดเดาหรือการสุ่มเนื้อหาขึ้นมาลอย ๆ (hallucination) ในบริบทของ [[compound-engineering]] การทำ grounding หมายถึงการระบุที่มาของทุกไอเดียที่ agent เสนอ

## แก่นความคิด

เมื่อ AI agent นำเสนอแนวทางแก้ปัญหา (เช่น ในคำสั่ง `/ce-ideate`) มนุษย์มักจะสงสัยว่า "ไอเดียนี้มาจากไหน?" หรือ "มันใช้ได้จริงกับโปรเจกต์เราไหม?"

**การทำ Grounding จะติดป้ายกำกับ (tag) ให้กับแต่ละไอเดียตามที่มาดังนี้:**

-   **Code Evidence:** มีหลักฐานยืนยันจากโค้ดที่มีอยู่จริงในโปรเจกต์ (เช่น "เสนอให้ใช้ function นี้เพราะเห็นว่ามีประกาศไว้ใน `utils.ts`")
-   **External Prior Art:** อ้างอิงจากแนวทางที่เป็นมาตรฐานสากลหรือโปรเจกต์อื่นที่เป็นที่ยอมรับ (Best practices)
-   **Reasoned Argument:** ข้อเสนอที่เกิดจากการใช้เหตุผลเชิงตรรกะจากบริบทหน้างาน (Logical deduction)

## ทำไปเพื่ออะไร?

เป้าหมายคือการผลักดันให้ agent เป็น **"Specifically wrong or specifically right"**

-   ถ้า agent เสนอสิ่งที่ "ผิด" เราจะเห็นทันทีว่ามันผิดที่จุดไหน (เช่น ผิดที่การตีความโค้ด หรือผิดที่ตรรกะ)
-   ลดการพูดแบบ "กั๊ก" (hedging) หรือการเสนอแบบกว้าง ๆ ที่ไม่มีประโยชน์
-   ช่วยให้มนุษย์ตัดสินใจเชื่อ (Trust) ได้ง่ายขึ้นเพราะเห็นที่มาที่ไป

## ผลคือ (The Payoff)

ช่วยลดอาการ **"vibecoding"** (การเขียนโค้ดตามความรู้สึกหรือ pattern ที่จำมาโดยไม่ดูบริบทจริง) และทำให้กระบวนการ brainstorm ระหว่างคนกับ AI มีประสิทธิภาพมากขึ้น
