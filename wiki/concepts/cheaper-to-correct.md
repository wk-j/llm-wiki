# Cheaper to Correct / แก้ให้ถูกได้ง่ายกว่า

"Cheaper to Correct" เป็นหลักการออกแบบ AI agent ที่เน้นการลด "ต้นทุน" (ในแง่ของเวลา ความพยายาม และความหงุดหงิด) เมื่อมนุษย์ต้องการแก้ไขสิ่งที่ agent เข้าใจผิดหรือทำผิด

## แก่นความคิด

ในวงการวิศวกรรมซอฟต์แวร์ เราทราบดีว่าการพบ bug ในขั้นตอนการออกแบบ (design) นั้น "ถูก" กว่าการพบ bug ในขั้นตอนการผลิต (production) หลายเท่าตัว

หลักการนี้ถูกนำมาใช้กับ AI agent ผ่านแนวคิดของ [[compound-engineering]] ว่า: **เราควรขยับการแก้ไขมาไว้ในขั้นตอนที่มันยังราคาถูกอยู่**

| ขั้นตอน | ลักษณะการแก้ไข | ต้นทุน (Cost) |
| :--- | :--- | :--- |
| **ก่อนเริ่มงาน** | แก้ไข bullet point ใน [[playback-pattern|playback]] | **ต่ำมาก** (แค่พิมพ์แก้ประโยคสั้น ๆ) |
| **ระหว่างงาน** | ตอบคำถาม scoping questions | **ต่ำ** (ให้ข้อมูลเพิ่ม) |
| **หลังจบงาน** | รื้อโค้ดที่ agent เขียนเสร็จแล้ว 10 ไฟล์ | **สูงมาก** (ต้องตรวจสอบโค้ดใหม่ทั้งหมด) |

## วิธีการทำให้เกิดขึ้น

1.  **Stop before acting:** อย่าปล่อยให้ agent วิ่งไปทำงานทันทีโดยไม่มีการ confirm (Show thinking).
2.  **Expose assumptions:** แบไต๋สิ่งที่ agent "เดา" ออกมาให้เห็นชัด ๆ (Inferred).
3.  **Low-friction feedback:** ทำให้ผู้ใช้สามารถแก้ไขสิ่งที่ผิดได้ด้วยการโต้ตอบสั้น ๆ ไม่ต้องสั่งงานใหม่ทั้งหมด

## ผลคือ (The Payoff)

เมื่อการแก้ไข "ราคาถูก" ผู้ใช้จะกล้าสั่งงาน agent มากขึ้น และกล้าปล่อยให้ agent ทำงานที่ซับซ้อนขึ้น เพราะรู้ว่าถ้ามีอะไรผิดพลาด จะสามารถดึงกลับมาได้ทันโดยไม่เสียเวลาไปเปล่า ๆ นำไปสู่สภาวะ **"Faster to trust"**
