# Memory Drift / ความจำเพี้ยน

อาการที่ระบบ AI หรือ Agent ยึดถือข้อมูลหรือความเชื่อเก่าที่หมดอายุไปแล้ว (stale state) มาใช้ในการตัดสินใจ ทำให้ผลลัพธ์ที่ได้ไม่สอดคล้องกับความต้องการหรือสถานการณ์ปัจจุบัน

## แก่นความคิด

ในระบบความจำแบบ [[llm-knowledge-bases|LLM Wiki]] ทั่วไป ข้อมูลเก่าและข้อมูลใหม่มักถูกเก็บไว้ในระดับความสำคัญที่เท่ากัน เมื่อ Agent ดึงข้อมูลมาใช้งานอาจเกิดปัญหา:
1. **ดึงข้อมูลที่ขัดแย้งกันมาพร้อมกัน:** เช่น เจอทั้งเบอร์โทรศัพท์เก่าและใหม่ในหน้าเดียวกัน
2. **เลือกใช้ข้อมูลเก่าเพราะตรงกับคำค้นหา:** Agent อาจจะเลือกข้อมูลที่ "เหมือน" คำค้นหามากกว่า ทั้งที่มีข้อมูลใหม่ที่ "ถูกต้อง" กว่าแต่ใช้คำต่างไป

## ทำไมถึงสำคัญ?

Memory Drift ไม่ใช่แค่เรื่องของความรก (clutter) แต่เป็นปัญหาด้าน **Reliability (ความน่าเชื่อถือ)** หาก Agent จำไม่ได้ว่าผู้ใช้เปลี่ยนใจไปแล้ว หรือจำค่าคอนฟิก (config) ผิด อาจส่งผลเสียร้ายแรงต่อระบบงานได้ โดยเฉพาะใน Agent ที่ทำงานต่อเนื่องเป็นเวลานาน

## แนวทางแก้ไข

- **[[memory-scoring|Scoring]] (การให้คะแนน):** ให้คะแนนข้อมูลตามความสดใหม่ (Freshness)
- **Conflict Resolution (การแก้ปัญหาความขัดแย้ง):** ตั้งกฎว่าข้อมูลใหม่ต้องชนะข้อมูลเก่า (Newer wins)
- **[[memory-decay|Memory Decay]] (การเลือนลางของความจำ):** การทำให้ข้อมูลเก่าค่อยๆ จางหายไปหรือถูกย้ายไปเก็บที่อื่นเพื่อไม่ให้รบกวนการทำงานปัจจุบัน
