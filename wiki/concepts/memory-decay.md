# Memory Decay / การเลือนลางของความจำ

กลไกที่ทำให้ความจำหรือข้อมูลในระบบของ Agent ค่อยๆ ลดความสำคัญลงตามกาลเวลา หรือถูกย้ายออกจากระบบประมวลผลหลักเมื่อไม่ได้ถูกใช้งานเป็นเวลานาน

## กลไกการทำงาน

เพื่อให้ระบบความจำไม่บวมและยังคงความสดใหม่อยู่เสมอ ระบบจะใช้ขั้นตอนเหล่านี้:
- **Weakening (การจางลง):** ข้อมูลที่ไม่ได้ถูกเรียกใช้นานๆ จะโดนลดคะแนน [[memory-scoring|Memory Scoring]] ลงเรื่อยๆ ตามธรรมชาติ
- **Archiving (การเก็บเข้ากรุ):** เมื่อคะแนนลดลงถึงจุดหนึ่ง ข้อมูลจะถูกย้ายจาก "Active Memory" (ความจำที่ใช้งานอยู่) ไปยัง "Cold Storage" (คลังเก็บข้อมูลสำรอง) ซึ่งอาจจะเข้าถึงยากกว่าแต่ไม่รบกวนการทำงานปัจจุบัน
- **Expiration (การหมดอายุ):** ข้อมูลบางประเภทที่มีกำหนดเวลาชัดเจน (เช่น แผนการทำงานในวันนี้) จะถูกลบทิ้งหรือระบุว่าหมดอายุอัตโนมัติเมื่อพ้นกำหนด

## ผลลัพธ์

ช่วยให้ Agent ไม่จมกองข้อมูล (Information overload) และรักษาความแม่นยำในการดึงข้อมูลที่เกี่ยวข้องจริงๆ ในปัจจุบันมาใช้งาน ทำให้ Agent ทำงานได้อย่างมีประสิทธิภาพแม้จะรันต่อเนื่องเป็นเวลานาน
