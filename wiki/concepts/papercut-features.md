# Papercut Features / ฟีเจอร์เก็บตกความรำคาญเล็กๆ

**Papercut Features** คือการแก้ไขบั๊กเล็กๆ หรือการเพิ่มฟีเจอร์จุกจิกที่อาจจะไม่ได้ส่งผลต่อฟังก์ชันหลักของโปรแกรม แต่ถ้าไม่มีจะสร้างความรำคาญ (friction) ให้กับผู้ใช้เปรียบเสมือนโดน "กระดาษบาด" (paper cut) ที่แม้แผลจะไม่ใหญ่แต่ก็สร้างความเจ็บปวดและน่ารำคาญได้ตลอดเวลา

## ลักษณะของ Papercut Features
- **ความสำคัญ:** มักจะมีความสำคัญต่ำใน Roadmap ของบริษัทใหญ่ (Low Priority)
- **ความรู้สึกผู้ใช้:** "ทำไมแค่นี้ทำไม่ได้?" หรือ "ถ้ามีตรงนี้จะสะดวกขึ้นมาก"
- **ตัวอย่าง:** 
    - การกดปุ่มลูกศรเพื่อเลื่อน cursor ในช่อง input ของ terminal (Caret-aware editing)
    - การจำตำแหน่งหน้าต่างล่าสุดที่เปิด
    - การมี shortcut key สำหรับฟังก์ชันที่ใช้บ่อยแต่ซ่อนอยู่ในเมนู

## ทำไมถึงสำคัญสำหรับโปรเจกต์ขนาดเล็ก?
[[panutat-tejasen|Panutat Tejasen]] ชี้ให้เห็นว่าการเก็บตก Papercut Features เป็นจุดแข็งที่ทำให้ [[thclaws]] แตกต่างจากผลิตภัณฑ์ยักษ์ใหญ่:
1. **Differentiator:** ผลิตภัณฑ์ใหญ่ๆ มักมุ่งเน้นที่ฟีเจอร์หลัก (Headline features) เพื่อตอบโจทย์คนหมู่มาก จนละเลยรายละเอียดเล็กๆ ที่ผู้ใช้ Power user หรือ Niche user ให้ความสำคัญ
2. **User Loyalty:** การที่ผู้ใช้ขอฟีเจอร์เล็กๆ แล้วได้รับการแก้ไขทันที สร้างความรู้สึกว่า "โปรเจกต์นี้ฟังเสียงเราจริงๆ"
3. **Polish:** แม้จะเป็นโปรเจกต์ Open-source หรือโปรเจกต์ขนาดเล็ก แต่ถ้ามีการเก็บรายละเอียด Papercut ที่ดี จะทำให้โปรแกรมดูมีคุณภาพ (Polished) และน่าใช้งานกว่าโปรแกรมใหญ่ที่ขาดความใส่ใจ

## Why this helps
การให้ความสำคัญกับ Papercut Features ช่วยลด **Cognitive Load** และความรำคาญสะสมของผู้ใช้ ทำให้ workflow การทำงานลื่นไหล (Flow state) โดยไม่ต้องสะดุดกับเรื่องเล็กน้อยที่ขัดใจ
