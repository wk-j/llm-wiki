# Non-Diegetic UI

UI elements that exist entirely outside the game world. Only the player can see them; characters are unaware of their existence. This is the most traditional and widespread form of game UI.

## Characteristics

- Rendered as 2D overlays on the screen, typically at edges or corners
- Displays information players need constant access to: health, score, ammo, time, resources
- Most explicit and accessible way to communicate game state
- Best suited for novice players who need clear, unambiguous feedback

## Common Examples

- Health bars, stamina bars, score counters
- Minimaps, compasses, quest markers
- Menu screens, inventory windows, quest logs
- Pop-up text, item displays, collectible indicators

## Trade-offs

| Strength | Limitation |
|---|---|
| Highly accessible and explicit | Can break immersion |
| Familiar conventions players understand | Screen clutter if overused |
| Works for all player skill levels | Placement and sizing require care |

## Relationship to Other Types

Non-diegetic UI is the opposite of [[diegetic-ui]] — it makes no attempt to justify its presence in the story world. Some games let players toggle diegetic elements to non-diegetic views for readability (e.g., expanding an in-world map to a full-screen overlay).
