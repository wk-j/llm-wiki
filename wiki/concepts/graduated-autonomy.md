# Graduated Autonomy (L1–L4) / ระดับอิสระของ AI agent 4 ชั้น

เมื่อเราให้ AI agent ทำงานที่มีความเสี่ยงสูง เช่น การทำ pentest (ทดสอบเจาะระบบ) คำถามสำคัญคือ "มันสามารถทำงานได้เองแค่ไหน และต้องมีคนควบคุมใกล้ชิดเพียงใด"

[[owasp-apts|OWASP APTS]] เสนอให้แบ่งระดับความเป็นอิสระของ AI agent ออกเป็น 4 ระดับ แทนที่จะมองแค่ว่า autonomous หรือไม่ ยิ่งระดับสูงขึ้น, agent ยิ่งทำงานได้เองมากขึ้น และต้องมีมาตรการป้องกันที่รัดกุมมากขึ้นตามไปด้วย

กรอบความคิดนี้ยังสามารถนำไปใช้กับ coding agent อื่นๆ ได้ ไม่ใช่แค่ pentest ตัวอย่างเช่น [[auto-mode]] ของ [[claude-code|Claude Code]] ก็เปรียบได้กับการเลื่อนระดับจาก L1 ไป L2 ในงานพัฒนาซอฟต์แวร์

## 4 ระดับคืออะไร

| ระดับ | ชื่อ | บทบาทของคน | Agent ทำเองได้แค่ไหน |
|---|---|---|---|
| **L1** | Assisted | สั่งงานทีละคำสั่ง | ใช้เทคนิคตามที่คนบอก ไม่สามารถทำขั้นตอนต่อไปเองได้ |
| **L2** | Supervised | อนุมัติเมื่อจะข้าม phase | สามารถรันหลายเทคนิคต่อเนื่องกันได้ภายใน phase เดียว |
| **L3** | Semi-Autonomous | กำหนดขอบเขตไว้ล่วงหน้า และเข้ามาเมื่อเกิดความผิดปกติ | สามารถรัน attack chain ที่ยาวจนจบได้ หากยังอยู่ในขอบเขตที่กำหนด |
| **L4** | Autonomous | รีวิวเป็นรายสัปดาห์หรือรายเดือน | ควบคุมหลายเป้าหมายพร้อมกัน และสามารถขยายขอบเขตได้เอง |

### Phase ของ pentest คืออะไร

ในมาตรฐาน APTS จะใช้ลำดับ phase ดังนี้:

```
Recon → Enumeration → Identification → Exploitation → Post-Exploitation → Reporting
```

แปลอย่างง่ายคือ: หาข้อมูลเป้าหมาย → สำรวจรายละเอียด → หาช่องโหว่ → เจาะระบบ → ดำเนินการหลังเจาะระบบ → เขียนรายงาน

L2 สามารถทำงานได้ใน phase เดียว แต่ต้องขออนุมัติจากคนเมื่อจะข้าม phase ในขณะที่ L3 สามารถข้าม phase ได้เอง

### ทำไมไม่ใช้แค่เปิด/ปิด

การแบ่งเป็น 4 ระดับมีเหตุผลด้านความเสี่ยง ซึ่งไม่ได้เพิ่มขึ้นอย่างค่อยเป็นค่อยไป แต่จะเพิ่มขึ้นตามช่วงเวลาที่ไม่มีคนควบคุม ยิ่งช่วงเวลายาวนาน ความเสี่ยงก็ยิ่งสูง

ยิ่ง agent ทำงานต่อเนื่องยาวนานโดยไม่มีคนคั่นกลาง โอกาสเกิดข้อผิดพลาดก็ยิ่งสูง การแบ่งเป็นระดับช่วยให้เรารู้ว่าจุดไหนที่ต้องมีคนเข้ามาเกี่ยวข้อง และจุดไหนที่สามารถปล่อยให้ทำงานเองได้ ซึ่งดีกว่าการถกเถียงกันว่า "autonomous" หมายถึงอะไร

## Tier กับ Level ไม่ใช่เรื่องเดียวกัน

สองคำนี้มักสร้างความสับสนใน APTS:

-   **Tier** บอกว่า platform ผ่านข้อกำหนดชุดใด (Tier 1, 2, 3) ซึ่งเป็นเรื่องของเครื่องมือ
-   **Level** บอกว่าในการทำงานจริง agent สามารถทำงานได้เองมากแค่ไหน (L1–L4) ซึ่งเป็นเรื่องของแต่ละงาน

โดยคร่าวๆ Tier 1 สามารถรันได้แค่ L1, Tier 2 สามารถไปถึง L3, และ Tier 3 จึงจะสามารถรัน L4 ได้ แต่ก็ไม่เสมอไป

ที่สำคัญคือ platform Tier 3 ก็ยังสามารถรันแบบ L1 ได้หากลูกค้าต้องการ ไม่ใช่ว่าซื้อ Tier 3 แล้วต้องรัน L4 ตลอดไป

## สิ่งที่ต้องเพิ่มเมื่อเลื่อนระดับ

การเลื่อนระดับไม่ใช่แค่การกดปุ่ม แต่ต้องมีมาตรการป้องกันใหม่ๆ เพิ่มขึ้นมาด้วยเสมอ

**จาก L1 ขึ้น L2:**
-   ต้องมี gate ให้คนอนุมัติเมื่อ agent จะข้าม phase
-   log ต้องละเอียดขึ้น สามารถเห็นได้ทีละขั้นตอน
-   เมื่อ agent เสนอ action, operator ต้องสามารถแก้ไขได้ก่อนที่จะดำเนินการ

**จาก L2 ขึ้น L3:**
-   กำหนดกลุ่ม action ที่อนุมัติไว้ล่วงหน้าให้ชัดเจน
-   มีระบบเฝ้าระวังขอบเขตอย่างต่อเนื่อง ไม่ใช่แค่ตอนเริ่มต้น
-   มี trigger แจ้งเตือนคนเมื่อเกิดความผิดปกติ

**จาก L3 ขึ้น L4:**
-   สามารถจัดการ campaign หลายเป้าหมายได้โดยไม่ต้องมีคนเข้ามาแทรกแซง
-   มีกฎเกณฑ์ในการค้นหาเป้าหมายใหม่ เพื่อไม่ให้ขยายขอบเขตเองโดยไม่มีการควบคุม
-   ต้องมี kill switch ที่มีคนพร้อมรับสาย 24/7
-   มีการรีวิวเป็นรอบตามกำหนด (รายสัปดาห์, รายเดือน, รายไตรมาส, รายปี)

สำหรับ L3 และ L4 ยังต้องมีการทำ containment verification (APTS-AL-028) เป็นระยะ เพื่อทดสอบว่า sandbox ยังสามารถจำกัดการทำงานของ agent ได้จริง

ข้อดีของแนวทางนี้คือ เมื่อลูกค้าถามว่า "เราสามารถขึ้นไป L3 ได้หรือไม่" แทนที่จะตอบแค่ใช่/ไม่ใช่ เราสามารถส่ง checklist ให้ตรวจสอบว่ายังขาดอะไรบ้าง และเมื่อครบถ้วนแล้วจึงสามารถเลื่อนระดับได้

## ความเชื่อมโยงกับแนวคิดอื่นใน wiki

-   [[auto-mode]] ของ Claude Code ทำงานคล้ายกับการเลื่อนจาก L1 → L2 แต่ใน layer ที่ต่างกัน (harness vs policy)
-   [[subagent-patterns|Pipeline subagents]] เป็นวิธีที่ช่วยให้ agent หลักสามารถไปถึง L2–L3 ได้โดยไม่ต้องมีคนคอยดูตลอดเวลา โดยใช้ subagent อื่นมาทำหน้าที่แทน ซึ่ง [[panutat-tejasen|Panutat]] มองในมุมเดียวกันและเรียกว่า [[harness-engineering]]
-   [[advisor-strategy]] มี logic คล้ายกับการ escalation ของ L3 แต่มีเป้าหมายต่างกัน (ประหยัดค่าใช้จ่าย vs ความปลอดภัย)
-   [[delegation-mindset]] ในการตัดสินใจว่าจะให้ [[claude-opus-4-7|Opus 4.7]] ทำอะไรใน turn เดียว ก็เปรียบเสมือนการเลือกระดับ L ในเวอร์ชันย่อ

## สรุป

เมื่อมี 4 ระดับ, คำถาม "AI agent ปลอดภัยหรือไม่" จะกลายเป็น 3 คำถามที่ตอบได้:

1.  ปัจจุบันรันอยู่ที่ระดับใด
2.  มีมาตรการป้องกันของระดับนั้นครบถ้วนหรือไม่
3.  หากจะเลื่อนระดับ ต้องเพิ่มอะไรบ้าง

ซึ่งทำให้ vendor, operator, และ CISO สามารถสื่อสารกันด้วยภาษาเดียวกันได้ โดยไม่ต้องถกเถียงเรื่องนิยามของคำว่า autonomous
