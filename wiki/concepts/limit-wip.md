# Limit WIP / การจำกัดงานในมือ

หลักการ **"Stop Starting, Start Finishing"** — การจำกัดจำนวนงานที่กำลังทำอยู่ (Work In Progress) ไม่ให้มากเกินความสามารถของระบบ

## ความสำคัญในยุค AI
เมื่อเรามี AI ช่วยให้ "เริ่ม" งานใหม่ได้ง่ายและเร็วมาก ความเสี่ยงคือเราจะเริ่มงานใหม่ทิ้งไว้เต็มไปหมดจนไม่มีงานไหน "จบ" จริงๆ

## กฎการปฏิบัติ
- **อย่าเริ่มงานใหม่ถ้างานเก่าติด:** ถ้างานเก่ายังติดอยู่ที่ Code Review หรือ Testing ให้เข้าไปช่วยเพื่อนเคลียร์งานนั้นก่อน แทนที่จะเอา AI มาปั่นงานใหม่เพิ่ม
- **Focus on Flow:** การจำกัด WIP ช่วยสร้างจังหวะ (Pace) ที่สม่ำเสมอทั้งทีม ลดการสลับงานไปมา (Context Switching)
- **ทลายคอขวด:** WIP ที่ต่ำทำให้เราเห็น [[theory-of-constraints|คอขวด]] ได้ชัดเจนขึ้น
