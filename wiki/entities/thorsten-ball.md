# Thorsten Ball

วิศวกรซอฟต์แวร์ที่ [[sourcegraph|Sourcegraph]] ทำงานกับทีม [[amp|Amp]] (coding agent ของ Sourcegraph) เป็นที่รู้จักในวงการก่อนหน้านี้จากหนังสือสอนเขียน interpreter/compiler ในภาษา Go (Monkey-lang) และจาก blog ส่วนตัว thorstenball.com ที่เขียนเรื่องวิศวกรรมและการอ่าน code

บน X ใช้ handle [@thorstenball](https://x.com/thorstenball) — โพสต์ของเขาในวงการ AI engineering มักมีน้ำหนักเพราะมาจากทั้งฝั่งคนเขียน infra ของ compiler/interpreter เอง และฝั่งคนสร้าง coding agent ที่ใช้งานจริง

## ผลงานสำคัญที่อ้างในวิกินี้

- [[software-after-software]] (2026-05-26) — 12-point manifesto จาก [[amp-labs|Amp Labs]] เรื่อง code abundance, process drag, frontier camp, และการ reorganize ทีมรอบ model

## จุดยืน

Ball อยู่ฝั่งที่เชื่อแรงว่าการเปลี่ยนผ่านครั้งนี้เป็นเรื่องระดับ "death of software development as we know it" ไม่ใช่ incremental improvement ใน Software After Software เขาเน้นว่า:

- Model ยังถูกประเมินต่ำ
- Code review จะหายไป ย้ายไป review decisions แทน
- ทีมเล็กที่จัดรอบ model จะชนะทีมใหญ่ที่ใส่ AI เข้า process เดิม
- ทุก institution ต้องมี frontier team

ข้อนี้แตกต่างจากเสียงเตือนของ [[lars-faye|Lars Faye]] เรื่อง [[cognitive-debt|cognitive debt]] หรือ [[matt-pocock|Matt Pocock]] เรื่อง [[specs-to-code|specs-to-code trap]] ที่ระวังการให้ leash ยาวกับ model
