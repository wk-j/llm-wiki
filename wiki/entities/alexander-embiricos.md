# Alexander Embiricos

**Alexander Embiricos** คือหัวหน้าฝ่ายผลิตภัณฑ์ของ [[openai-codex|Codex]] ที่ [[openai|OpenAI]] — co-present keynote ใน [[ai-engineer-worlds-fair|AI Engineer World's Fair]] คู่กับ [[romain-huet|Romain Huet]] (สรุปใน [[piyalitt-codex-keynote-attention-not-token]])

## มุมมอง product ที่ชัด

- ของโปรดในปี 2026: **Codex app**, **Goal Mode**, **Remote** — และทั้งหมดเกิดจากทีมใช้ Codex สร้าง Codex เอง
- agent ทำได้แทบทุกอย่างบนคอมพิวเตอร์ — ช่วยก่อนเขียน code (ทำไมงานนี้ต้องเกิด) และหลัง (review, deploy)
- สิ่งที่น่าตื่นเต้นไม่ใช่แค่ความเร็ว แต่คือ **ตัดสินใจได้ดีขึ้นว่าควรสร้างอะไร** — prototype หลายไอเดีย เหลือเวลาคลุก user จริง
- เป้าหมายไม่ใช่แทนที่ engineer แต่เสริมพลัง — ผู้ใช้ต้องรู้สึกเชี่ยวชาญและเป็นเจ้าของงานเสมอ

## Codex app — จากโดนค้านสู่ผู้ใช้

Alexander เล่าเบื้องหลังว่า Codex app เคยถูกค้านหนักตั้งแต่ pitch — developer บอกไม่มีวันออกจาก terminal/Vim. วันนี้คนกลุ่มเดียวกันกลายเป็นผู้ใช้. เหตุผล:

- CLI เป็นหน้าจอทำงานร่วมกันสำหรับทุกประเภทงานไม่ได้
- IDE บังคับเริ่มจาก code ทั้งที่งานควรเริ่มจากคุย
- Codex app = chat เรียบง่าย + ลงลึกเมื่อจำเป็น

เขาอ่านนิยายวิทยาศาสตร์เยอะ และมองว่า **chat ถูกประเมินค่าต่ำเกินไป** — โลกต้องการ chat เป็นตัวตนเดียวคู่กับหน้าจอทำงานร่วมกันทรงพลัง
