# Alexander Embiricos

**Alexander Embiricos** คือหัวหน้าฝ่ายผลิตภัณฑ์ของ [[openai-codex|Codex]] ที่ [[openai|OpenAI]]. ใน keynote ที่ [[ai-engineer-worlds-fair|AI Engineer World's Fair]] เขาขึ้นเวทีคู่กับ [[romain-huet|Romain Huet]] เพื่อเล่าว่า Codex กำลังขยับจากเครื่องมือเขียน code ไปเป็นพื้นที่ทำงานร่วมกับ agent อย่างไร (สรุปใน [[piyalitt-codex-keynote-attention-not-token]])

## มุมมอง product ที่ชัด

Alexander ยกสามอย่างที่เขาชอบในปี 2026: **Codex app**, **Goal Mode**, และ **Remote**. จุดที่เขาเน้นคือทั้งหมดเกิดจากทีมใช้ Codex สร้าง Codex เอง ไม่ใช่แค่ demo ขายของ แต่เป็นเครื่องมือที่ทีมใช้กับงานจริง

มุม product ของเขาชัดมาก: agent ไม่ได้ช่วยเฉพาะตอนเขียน code แล้ว แต่เริ่มช่วยตั้งแต่ก่อนเขียน code เช่น ทำไมงานนี้ต้องเกิด ไปจนถึงหลังเขียน เช่น review และ deploy. สิ่งที่น่าตื่นเต้นจึงไม่ใช่แค่ทำงานเร็วขึ้น แต่คือ **ตัดสินใจได้ดีขึ้นว่าควรสร้างอะไร** เพราะทีม prototype ได้หลายทางและมีเวลาคุยกับ user จริงมากขึ้น

เป้าหมายของ Codex ในกรอบนี้ไม่ใช่แทนที่ engineer แต่เสริมพลังให้ engineer. ผู้ใช้ยังต้องรู้สึกว่าเชี่ยวชาญและเป็นเจ้าของงานเสมอ

## Codex app — จากโดนค้านสู่ผู้ใช้

Alexander เล่าเบื้องหลังว่า Codex app เคยถูกค้านหนักตั้งแต่ตอน pitch. Developer หลายคนบอกว่าไม่มีวันออกจาก terminal/Vim. แต่วันนี้คนกลุ่มเดียวกันกลับกลายเป็นผู้ใช้. เหตุผลคือ:

- CLI ไม่ใช่หน้าจอทำงานร่วมกันที่เหมาะกับงานทุกประเภท
- IDE มักบังคับให้เริ่มจาก code ทั้งที่หลายงานควรเริ่มจากการคุยให้เข้าใจก่อน
- Codex app เป็น chat ที่เรียบง่าย และค่อยลงลึกเมื่อจำเป็น

เขาอ่านนิยายวิทยาศาสตร์เยอะ และมองว่า **chat ถูกประเมินค่าต่ำเกินไป**. โลกต้องการตัวช่วยที่คุยด้วยได้ต่อเนื่องในหลายงาน คู่กับหน้าจอทำงานร่วมกันที่ทรงพลังพอสำหรับตอนต้องตรวจและปรับทิศ
