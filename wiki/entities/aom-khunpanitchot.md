# Aom Khunpanitchot / ออม คุณพานิชโชติ

[[aom-khunpanitchot|Aom Khunpanitchot]] คือผู้โพสต์ Facebook ใน source [[aom-fable-elysia-2-audit|Fable audit Elysia 2]]. Capture ที่ให้มาไม่มี author field ชัด ๆ แต่ URL อยู่ใต้ path `AomKhunpanitchot` จึงบันทึกเป็นชื่อ entity แบบระมัดระวัง.

ในโพสต์นี้ Aom เล่าจากมุมคนดูแล [[elysia-2|Elysia 2]] มานานหลายปี และใช้ AI หลายตัว audit code ก่อนตัดสินใจเรื่อง RC/stable release. จุดที่น่าสนใจคือเขาไม่ได้พูดในฐานะคนตื่นเต้นกับ demo ทั่วไป แต่ในฐานะ owner ของ codebase ที่มี mental model ของระบบอยู่แล้ว.

## มุมมองจาก source นี้

- ใช้ AI review หลายตัวกับ Elysia 2 แล้วส่วนใหญ่บอกว่าพร้อม stable/RC
- ทดลอง [[fable|Fable]] ด้วย prompt สั้นกว่า แต่ปล่อย auto mode รันยาวและแตก agent จำนวนมาก
- มองว่า Fable สร้าง report ได้ละเอียดเกินระดับที่มนุษย์ทั่วไปจะยอมลงทุน audit
- ยังเชื่อว่าคนมีบทบาทในงาน domain expert, specialist, tech lead, และ decision-making แต่คนที่ทำแค่ coding ตามสั่งอาจเสี่ยงขึ้น

**ได้อะไร:** Aom เป็นตัวอย่างของ developer-owner ที่ใช้ AI ไม่ใช่แค่เร่ง implementation แต่ใช้เป็นตัวกด release judgement ให้เข้มขึ้น.
