# Alex Ker

วิศวกรที่เขียนบทความเกี่ยวกับ agent harness. Twitter handle **@thealexker**; GitHub handle **AlexKer**. หน้า entity นี้ถูกจัดเก็บภายใต้ชื่อสไตล์ GitHub เพื่อให้สอดคล้องกับธรรมเนียมของ `panutat-tejasen` / `nattee-niparnan`

## สิ่งที่แหล่งข้อมูลยืนยัน

- เป็นผู้ใช้งานและผู้มีส่วนร่วมใน open-source harness: [Roo Code](https://roocode.com/), [DeepAgent CLI](https://docs.langchain.com/oss/python/deepagents/cli/overview), [[humanlayer|HumanLayer]]
- เคย/ปัจจุบันทำงานที่ [[baseten|Baseten]] — สร้าง **gpt-oss-swarm** ตอนที่ `gpt-oss-120b` เปิดตัว (อ้างอิงที่ [github.com/AlexKer/gpt-oss-swarm](https://github.com/AlexKer/gpt-oss-swarm)) ซึ่งเป็นต้นแบบของ subagent แบบ parallel-fan-out
- เผยแพร่บทความ (18 เม.ย. 2026) *"Harnesses are everything. Here's how to optimize yours."* — ซึ่งสังเคราะห์คำแนะนำในการปรับแต่ง harness ครอบคลุมทั้ง progressive disclosure, R.P.I. prompting, และ subagent patterns

## จุดยืน

Harness — ไม่ใช่ model — คือที่ที่วิจารณญาณทางวิศวกรรม (engineering judgment) สถิตอยู่ในการเขียนโค้ดด้วย AI เขาเปรียบเทียบว่า: "วิศวกรเคยเถียงกันเรื่อง IDEs, ตอนนี้เราเถียงกันเรื่อง harnesses" มองว่า harness เป็นชิ้นส่วนโครงสร้างพื้นฐานของทีมที่สั่งสมความรู้ขององค์กร; การเปลี่ยน harness เท่ากับเป็นการรีเซ็ตความรู้นั้น

เขามักจะอ้างอิงแนวคิดจาก [[humanlayer|HumanLayer]] (Kyle) อยู่บ่อยครั้ง: "dumb zone" ที่เกี่ยวกับ instruction budget, และ R.P.I. framework
