# Vtrivedy

ผู้ใช้ X `@Vtrivedy10` ที่ [[agent-harness-engineering]] อ้างว่าเป็นคน coin คำว่า **harness engineering** ในความหมายของการแยก model ออกจาก scaffolding รอบ ๆ มัน

## ความสำคัญ

นิยามที่ Addy ยกมาคือ:

> "Agent = Model + Harness. If you're not the model, you're the harness."

ประโยคนี้ทำให้คำว่า harness ชัดขึ้น: ทุกอย่างที่ไม่ใช่ตัว model แต่ทำให้ model กลายเป็น agent ได้ นับเป็น harness ทั้งหมด เช่น prompt, tool, context policy, sandbox, memory, hooks, และ orchestration loop
