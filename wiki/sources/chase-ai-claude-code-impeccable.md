# Claude Code + Impeccable = Design CHEAT CODE

[[chase-ai|Chase AI]] demonstrates how to use [[impeccable|Impeccable]], an open-source design skill/tool for [[claude-code|Claude Code]], to improve the visual quality of AI-generated frontend designs and eliminate "AI slop."

## Key Takeaways

- **The Problem of "AI Slop":** AI-generated designs often look mediocre because users don't use professional design terminology. Claude Code, by default, lacks a sophisticated design vocabulary.
- **Impeccable as a Solution:** Impeccable is a GitHub repository that acts as a comprehensive "skill" for Claude Code. It includes 23 commands and 7 domain-specific reference files.
- **Seven Pillars of Design:** Impeccable focuses on Typography, Color, Spatial Design, Responsiveness, Interactions, Motion, and UX Writing.
- **Impeccable Live (Alpha):** A browser-based interface that allows for live editing, variant testing, and micro-adjustments directly on the webpage while synced with Claude Code.
- **Core Workflow:**
    1. **Greenfield:** Use `impeccable craft` to start an interview-driven design process.
    2. **Refinement:** Generate multiple variants (macro layouts) to compare.
    3. **Live Editing:** Use Impeccable Live to tweak components with commands like `bolder`, `delight`, or `polish`.
    4. **Auditing:** Use `impeccable critique` to get a design health score and identify anti-patterns (e.g., "decoration discipline" issues).

## Key Commands

- `impeccable craft`: Starts the initial design interview and sets up `product.md` and `design.md`.
- `impeccable live`: Launches the alpha browser-based editing mode.
- `impeccable document`: Reverse-engineers a `design.md` file from an existing codebase.
- `impeccable critique`: Audits the site for "AI slop" and design violations.
- `impeccable polish`: Performs a final design system pass.
- `impeccable bolder`: Pushes designs toward higher impact without losing clarity.
- `impeccable delight`: Adds personality to components.
