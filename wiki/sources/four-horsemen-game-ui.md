# The Four Horsemen of Game UI Design

Corporation Pop article classifying video game UI into four types based on two axes: whether the element exists in the game world, and whether the character is aware of it.

## The Four Types

| Type | In game world? | Character aware? | Example |
|---|---|---|---|
| **[[diegetic-ui\|Diegetic]]** | Yes | Yes | Dead Space health spine, in-world holograms |
| **[[non-diegetic-ui\|Non-diegetic]]** | No | No | Health bars, score counters, menus |
| **[[spatial-ui\|Spatial]]** | Yes | No | Pathfinding lines, enemy nameplates, interaction prompts |
| **[[meta-ui\|Meta]]** | No | Yes (implicitly) | Screen blood, blur, shake, dirt accumulation |

## Key Insights

- **Immersion vs. accessibility trade-off**: diegetic UI maximizes immersion but is the least accessible; non-diegetic is the most explicit and accessible but breaks immersion.
- **Spatial as middle ground**: spatial UI places elements in the 3D world for context without claiming the character sees them — balances usability and immersion.
- **Meta creates empathy**: meta UI links the player's perception to the character's physical/mental state through screen effects rather than explicit indicators.
- **Most games mix all four**: a single game rarely uses just one type; designers combine them to serve different information needs.
- **Audience matters**: diegetic and meta UI reward experienced players who can read subtle cues; non-diegetic and spatial are better for novices.
- **Toggleability**: good game UI design often lets players disable spatial/meta elements to suit their preference.
