# Somkiat — Model Choice by Domain Expertise

โพสต์สั้นบน Facebook ของ [[somkiat-khitwongwattana|Somkiat Khitwongwattana]] หรือ Akexorcist (2026-04-21) เสนอเกณฑ์ง่าย ๆ ในการเลือก model: ให้ *taste ของคน* เป็นตัวตัดสิน ไม่ใช่ hype ของ model ใหม่

## สรุปเนื้อหา

สองกรณี:

1. **ลงอ่านโค้ดเอง + เป็น Domain Expert อยู่แล้ว** — ใช้แค่ Claude Sonnet 4.5 ก็พอกับงานส่วนใหญ่ มีน้อยมากที่ต้องขยับไป model ใหม่กว่านี้
2. **Vibe — ไม่ได้ลงไปดูโค้ดเอง หรือไม่มีพื้นในเรื่องนั้นเลย** — เลือก model ใหม่ไว้ก่อนก็เข้าใจได้ เพราะ model ใหม่ให้ *ความมั่นใจในผลลัพธ์* มากกว่า

## Quote (verbatim, Thai)

> โดยส่วนตัวคิดว่าถ้าเป็นงานที่เข้าไปลงรายละเอียดในโค้ดและเราเป็น Domain Expert อยู่แล้ว ใช้แค่ Claude 4.5 Sonnet ก็เพียงพอกับงานส่วนใหญ่แล้วนะ มีน้อยมากที่ต้องใช้ Model ที่ใหม่กว่านี้
>
> แต่ถ้าเป็นงานที่ต้องการ Vibe แบบไม่ได้ลงไปดูโค้ดด้วยตัวเอง หรือไม่ได้มี Knowledge ในเรื่องนั้นเลย การเลือกใช้ Model ใหม่ ๆ ไว้ก่อน อันนี้ก็เข้าใจได้ เพราะยังไงก็ทำให้เรามั่นใจในผลลัพธ์มากกว่าการใช้ Model เก่าแน่นอน

## แก่น / ทำไมถึงน่าสนใจ

เกณฑ์นี้พลิก default ของข่าวเรื่อง model ใหม่: จาก **"model ใหม่ย่อมดีกว่า"** → **"model ใหม่ช่วย *กลบสิ่งที่คนยังไม่รู้* ได้เท่านั้น"**

- คนที่มี taste (Domain Expert) — ตัว *คน* นั่นแหละที่เป็นคอขวดของคุณภาพ; model ที่ใช้อยู่แล้วเพียงพอ
- คนที่ไม่มี taste (Vibe) — ตัว *model* ต่างหากที่เป็นคอขวด; ยอมจ่ายแพง / ใหม่กว่า เพื่อลดความเสี่ยง

ตรงกับ [[taste-paradox]] ของ [[nattee-niparnan|Nattee Niparnan]] จากอีกมุม — Nattee พูดว่า *taste เกิดจากการเคยฝึกมือเอง*; Somkiat ต่อว่า *เมื่อมี taste แล้วเครื่องมือถูก ๆ ก็พอ*

## ข้อควรระวัง

- **เป็น heuristic ของคนที่อยู่หน้างาน** ไม่ใช่ benchmark
- **Claude 4.5 Sonnet** ที่ Somkiat อ้างคือ [[claude|Claude Sonnet 4.5]] (ไม่ใช่ [[claude-opus-4-7|Opus 4.7]]) — ช่วงเวลาที่โพสต์ยังเป็น baseline "พอใช้งาน" สำหรับงาน dev
- **ไม่ได้แปลว่า Opus ไม่ดี** — แปลว่า *ส่วนที่ดีขึ้น* จาก model ใหม่จะคุ้มหรือไม่ ขึ้นกับว่าคนใช้มี taste พอจะ *รับ* ส่วนนั้นได้ไหม
