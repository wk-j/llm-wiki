# A Framework for Frontier AI and the Dawning of a New Age / กรอบกำกับ Frontier AI ก่อนเข้าสู่ยุคใหม่

บทความของ [[demis-hassabis|Demis Hassabis]] (นักวิจัย AI และผู้ร่วมก่อตั้ง DeepMind) วางภาพใหญ่สองด้านพร้อมกัน ด้านหนึ่ง เขาเชื่อว่า [[artificial-general-intelligence|Artificial General Intelligence หรือ AGI]] อาจมาในอีกไม่กี่ปีและสร้างประโยชน์มหาศาล อีกด้าน ความสามารถด้าน cyber, bio, nuclear และ agent ที่ปรับปรุงตัวเองอาจโตเร็วกว่าความเข้าใจของเรา

ข้อเสนอหลักคือให้สหรัฐฯ ตั้ง [[frontier-ai-standards-body|Frontier AI Standards Body]] เพื่อทดสอบ model ที่แตะระดับความเสี่ยงสูงก่อนปล่อย เริ่มจากความร่วมมือแบบสมัครใจ แล้วค่อยทำเป็นเงื่อนไขบังคับสำหรับการ deploy ในตลาดสหรัฐฯ เมื่อระบบทดสอบพิสูจน์ว่าใช้ได้จริง

นี่เป็นบทความเสนอทิศทาง ไม่ใช่ประกาศนโยบายหรือหลักฐานว่า AGI จะมาถึงตามเวลา ตัวเลข “10 เท่าของ Industrial Revolution ด้วยความเร็ว 10 เท่า” และภาพ post-scarcity เป็นวิสัยทัศน์ของผู้เขียน

## AGI อยู่ใกล้แค่ไหน

Hassabis นิยาม AGI ในบทความนี้ว่า “ระบบที่มีความสามารถทางความคิดทุกอย่างที่สมองมนุษย์มี” และคาดว่าอาจเหลือเวลาเพียงไม่กี่ปี เขาเรียกช่วงปัจจุบันว่า foothills of the singularity หรือเชิงเขาก่อนถึงจุดที่ความเปลี่ยนแปลงเร่งจนโลกเดิมตามไม่ทัน

เขาเปรียบ AGI กับการค้นพบไฟหรือไฟฟ้า มากกว่า internet หรือ mobile เพราะ AGI ไม่ได้เปลี่ยนแค่ช่องทางสื่อสาร แต่น่าจะเร่งการค้นพบในหลายสาขาพร้อมกัน ตัวอย่างที่ยกคือยา พลังงานสะอาด วัสดุใหม่ productivity และโลกที่ทรัพยากรอาจไม่ใช่ข้อจำกัดหลักอีกต่อไป

แต่บทความเดียวกันก็ยอมรับว่าไม่มีใครรู้แน่ว่าจะเกิดอะไรขึ้น ผู้เชี่ยวชาญยังเห็นไม่ตรงกัน จึงมีความตึงในตัวเอง: timeline ถูกพูดอย่างมั่นใจ แต่เส้นทางและผลลัพธ์ยังไม่แน่นอน

**ได้อะไร:** “AGI ในไม่กี่ปี” ควรเก็บเป็น forecast ของ Hassabis ไม่ใช่ countdown ที่วงการตกลงร่วมกันแล้ว

## ความเสี่ยงโตไปพร้อมประโยชน์

ผู้เขียนบอกว่า frontier AI เริ่มสร้างประโยชน์จริงแล้ว แต่ความสามารถใหม่ก็มาพร้อม risk ใหม่ เขาชี้ว่า cybersecurity เป็นสัญญาณที่เห็นแล้ว ส่วน nuclear และ biological threats อาจตามมาเมื่อ model เก่งขึ้น

ถัดจากนั้นคือ agentic system ที่ทำงานเองนานขึ้นและ recursively self-improve หรือช่วยปรับปรุง model/harness รอบถัดไป ระบบแบบนี้ต้องมี safeguard ที่คุมได้จริง รวมทั้งต้องยอมรับว่าอาจมีปัญหาบางชนิดที่เรายังตั้งชื่อไม่ได้

แรงกดดันสำคัญคือการแข่งขันหลายชั้น ทั้งบริษัทและภูมิรัฐศาสตร์ การแข่งช่วยเร่งประโยชน์ แต่ก็ทำให้ capability วิ่งเร็วกว่าความเข้าใจของนักวิจัยและนโยบาย Hassabis จึงเสนอ “cautious optimism”: เดินหน้าต่อ แต่สร้างเวลาสำหรับ test, security และความร่วมมือระหว่างประเทศ

**ผลคือ:** ทางเลือกในบทความไม่ใช่เร่งเต็มที่หรือหยุดทั้งหมด แต่คือทำให้ความรับผิดชอบเป็นแรงจูงใจที่วิ่งคู่กับ innovation

## โครงของ Frontier AI Standards Body

ข้อเสนอเริ่มจากสหรัฐฯ เพราะมีฐานเศรษฐกิจ เทคนิค หน่วยงานรัฐ และ National Labs พร้อมกว่า หน่วยมาตรฐานอาจเป็น public-private partnership ที่รัฐบาลกำกับ หรือ self-regulatory organisation คล้าย FINRA ในตลาดการเงิน

โครงสร้างที่เสนอมีองค์ประกอบสำคัญ:

- board มีผู้เชี่ยวชาญเทคนิคอิสระและตัวแทน open source
- เงินทุนจำนวนมากมาจากอุตสาหกรรม เพื่อจ้างคนระดับโลกและซื้อ compute สำหรับ test ใหญ่
- ทำงานกับหน่วยงานรัฐบาลและ US National Labs ในงานที่แตะ national security
- สร้าง protocol เพื่อจัด model เป็น `Frontier-class` ตาม threshold ที่อัปเดตตาม capability
- องค์กรที่มี model ถึง threshold ถูกเรียกว่า `Frontier Lab`

Frontier Lab ควรเผยแพร่ model card, รักษา internal cybersecurity, vet บุคลากรสำคัญ และลงทุนใน safety/security research ให้พอ การได้สถานะนี้ควรมี prestige เพื่อเปลี่ยนความรับผิดชอบจากภาระให้กลายเป็นสัญญาณคุณภาพ

**ได้อะไร:** ขอบเขตการกำกับผูกกับ capability ของ model ไม่ได้ผูกกับชื่อบริษัทหรือเทคโนโลยีชนิดเดียว

## จากสมัครใจ 30 วันไปสู่กติกาบังคับ

ระยะแรก Lab สมัครใจส่ง model ให้ตรวจล่วงหน้าสูงสุด 30 วันก่อน release พอ protocol พิสูจน์ว่าแม่นและทนต่อการเล่นเกม จึงค่อย formalise เป็นเงื่อนไขก่อน deploy Frontier Model ในสหรัฐฯ หลังปล่อยแล้ว Lab ยังต้องช่วยแก้ critical vulnerability ที่พบ

บทความเปิดทางให้เพิ่มความเข้มตามความเสี่ยง ถ้าสถานการณ์ร้ายแรง หน่วยมาตรฐานอาจประสาน Frontier Labs ให้ชะลอการพัฒนาพร้อมกัน การชะลอแบบประสานสำคัญ เพราะถ้าหยุดฝ่ายเดียว คู่แข่งอาจได้เปรียบและแรงจูงใจให้ร่วมมือจะพัง

อย่างไรก็ดี บทความยังไม่ลงรายละเอียดเรื่องอำนาจตามกฎหมาย การอุทธรณ์ liability หรือใครตัดสินว่า threshold ไหนร้ายแรงพอให้ชะลอ

**ผลคือ:** ทางเดินที่เสนอคือ `voluntary review → พิสูจน์ protocol → mandatory market gate → เพิ่มความเข้มเมื่อ risk สูงขึ้น`

## Test ต้องเปลี่ยนตาม model

การประเมินควรครอบคลุม cyber, biological threats และโดเมนความเสี่ยงสูงอื่น ๆ สำหรับ agent ควรทดสอบการพยายามข้าม safety guardrail กับสัญญาณ deception ผู้เขียนยังเสนอ watermark สำหรับภาพ AI และ human-readable output token เพื่อช่วยเข้าใจ reasoning

benchmark ต้องอัปเดตสม่ำเสมอ อาจเริ่มทุกไตรมาส ชุดที่อิ่มตัวหรือล้าสมัยต้องปลดออก ช่วงแรก Lab ช่วยออกแบบ test ได้ แต่ระยะยาว Standards Body ต้องสร้าง held-out test เอง เพื่อกัน Lab train หรือปรับ model ให้ชนะข้อสอบที่รู้ล่วงหน้า

รัฐบาลควรช่วยสร้าง ecosystem ของ third-party auditor ทั้งสำหรับตรวจ model และออกแบบ benchmark ใหม่ จุดนี้เชื่อมกับ [[benchmark-contamination|benchmark contamination]] และ [[reward-hacking|reward hacking]] โดยตรง: ถ้าเกณฑ์ผ่านตลาดผูกกับคะแนน model ก็มีแรงจูงใจสูงมากให้ optimize ตัวชี้วัดแทนลดความเสี่ยงจริง

**ได้อะไร:** test ไม่ใช่ checklist คงที่ แต่เป็นระบบที่ต้องหมุนข้อสอบ เก็บบางส่วนเป็นความลับ และตรวจว่าคะแนนสะท้อนพฤติกรรมจริง

## Open/closed และประเทศต้นทางไม่ควรเปลี่ยนเกณฑ์

กรอบนี้ตั้งใจใช้กับ Frontier-class model ไม่ว่าจะมาจากประเทศใดและเปิดหรือปิด weights ส่วน startup, academia หรือ model ที่ยังไม่ถึง threshold ได้รับการยกเว้น เพื่อไม่ให้ compliance cost ขวางนวัตกรรมตั้งแต่ต้น

การมีตัวแทน open source บน board ช่วยให้มุมของผู้พัฒนา model เปิดไม่หายไป แต่ยังมีคำถามยาก: model เปิดที่คนดาวน์โหลดไปแก้หรือ deploy นอกเขตอำนาจได้ จะผ่าน market gate และ post-release remediation แบบเดียวกับ closed API ได้อย่างไร

**ผลคือ:** Hassabis เสนอ risk-tiering ตาม capability แต่ implementation ต้องแยก distribution model ไม่งั้นกติกาเดียวกันอาจบังคับใช้ได้ไม่เท่ากัน

## จากกรอบสหรัฐฯ ไปสู่มาตรฐานโลก

ผู้เขียนมองว่าสหรัฐฯ ควรเริ่มก่อนแล้วใช้กรอบนี้เป็นฐานเจรจาระหว่างประเทศ เพราะผลกระทบของ AGI ข้ามพรมแดน เป้าหมายสุดท้ายคือมาตรฐานร่วมสำหรับ risk ที่ร้ายแรงที่สุด พร้อมรักษาการเข้าถึงประโยชน์ของ AI ให้กว้าง

หลังแก้โจทย์เทคนิคยังมีโจทย์สังคม: ระบบเศรษฐกิจแบบไหนเหมาะกับ post-scarcity, คนจะหา meaning และ purpose จากอะไร และคุณค่าของมนุษย์ควรเปลี่ยนอย่างไร Hassabis ย้ำว่าคำตอบเหล่านี้ปล่อยให้ technologist ตัดสินฝ่ายเดียวไม่ได้

**ได้อะไร:** governance ของ AGI ไม่จบที่ benchmark หรือ model card เพราะการกระจายประโยชน์และความหมายของชีวิตเป็นการตัดสินใจของสังคม

## ประเด็นที่ยังเปิดอยู่

- นิยาม “มีความสามารถทุกอย่างที่สมองมี” จะกลายเป็นเกณฑ์ AGI ที่วัดได้อย่างไร
- forecast อีกไม่กี่ปีสอดคล้องกับคำยอมรับว่าไม่มีใครรู้แน่และผู้เชี่ยวชาญเห็นต่างอย่างไร
- หน่วยที่รับเงินจากอุตสาหกรรมจะกัน regulatory capture และรักษาความเป็นอิสระได้อย่างไร
- threshold แบบ benchmark จะกัน teaching-to-the-test, leakage และ reward hacking ได้พอหรือไม่
- การอ่าน output token บอก reasoning ภายในของ model ได้จริงแค่ไหน
- กติกาเดียวกันจะบังคับใช้กับ closed API, open-weight model และ model จากต่างประเทศอย่างเท่าเทียมได้อย่างไร
- ใครมีอำนาจสั่ง slowdown และประเทศคู่แข่งจะยอมชะลอพร้อมกันด้วยกลไกอะไร
- ภาพ post-scarcity จะเกิดขึ้นจริงหรือประโยชน์จะกระจุกตัวจน scarcity เปลี่ยนรูปแทนที่จะหายไป
